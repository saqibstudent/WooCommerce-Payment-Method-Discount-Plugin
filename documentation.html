üì¶ Installation

Method 1: Manual Installation

Download the Plugin

Download the plugin files from your source and extract the ZIP file.

Upload to WordPress

Upload the wc_payment_discount_plugin folder to your /wp-content/plugins/ directory via FTP or hosting file manager.

Activate the Plugin

Go to Plugins ‚Üí Installed Plugins in your WordPress admin and activate the "WooCommerce Payment Method Discount" plugin.

Configure Settings

Navigate to WooCommerce ‚Üí Payment Discounts to configure your discount rules.

Method 2: WordPress Admin Upload

Access Plugin Upload

Go to Plugins ‚Üí Add New in your WordPress admin panel.

Upload Plugin File

Click Upload Plugin and choose the plugin ZIP file, then click Install Now.

Activate and Configure

Activate the plugin and configure via WooCommerce ‚Üí Payment Discounts.

Note: Make sure WooCommerce is installed and activated before installing this plugin.

Database Storage

Plugin settings are stored in WordPress options table:

wc_payment_discount_enabled: Plugin enable/disable status (0 or 1) wc_payment_discount_rules: Array of discount rules per payment method

Session Management

Payment method tracking uses WooCommerce sessions:

selected_payment_method: Currently selected payment method chosen_payment_method: Fallback payment method storage

AJAX Implementation

AJAX Flow

  • Trigger: Payment method radio button change
  • Action: update_payment_discount
  • Security: Nonce verification
  • Response: Triggers checkout update if discount applicable

üõ†Ô∏è Troubleshooting

Discount Not Applying

Symptoms: Discount doesn't appear when payment method is selected

Solutions:

  1. Verify plugin is enabled in WooCommerce ‚Üí Payment Discounts
  2. Check discount rules are properly configured with values > 0
  3. Ensure JavaScript is not blocked by other plugins or browser
  4. Clear browser cache and cookies
  5. Test with default WordPress theme to rule out theme conflicts
AJAX Errors

Symptoms: Console errors during payment method selection

Solutions:

  1. Check WordPress AJAX functionality is working
  2. Verify nonce generation and verification
  3. Temporarily disable conflicting plugins
  4. Check server error logs for PHP errors
  5. Ensure proper WordPress and WooCommerce versions
Incorrect Discount Calculation

Symptoms: Wrong discount amount applied

Solutions:

  1. Verify discount type setting (percentage vs. fixed amount)
  2. Check for cart total calculation conflicts
  3. Review tax calculation order and settings
  4. Ensure proper decimal/number formatting in admin
  5. Test with simple product to isolate complex cart scenarios
Plugin Conflicts

Symptoms: Plugin doesn't work with other checkout plugins

Solutions:

  1. Test with only WooCommerce and this plugin active
  2. Gradually reactivate other plugins to identify conflicts
  3. Check for checkout customization plugins that may interfere
  4. Review plugin load order if necessary
  5. Contact support with conflicting plugin details

Debug Mode

To enable WordPress debug mode for troubleshooting, add this to your wp-config.php file:

define('WP_DEBUG', true); define('WP_DEBUG_LOG', true); define('WP_DEBUG_DISPLAY', false);

Check /wp-content/debug.log for plugin-related errors.

Pro Tip: Always test changes on a staging site before applying to production.
